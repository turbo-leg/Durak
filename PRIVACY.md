# Privacy Policy

**Last updated:** 2026-05-16
**Service:** Durak Online — Discord Activity (the "Service")
**Operator contact:** tubulol12345@gmail.com

> This is a v1 draft prepared for Discord App Directory submission. It has not been reviewed by counsel; please consult a lawyer before relying on it for a production launch outside the EU/US "small game" risk profile.

This Privacy Policy explains what information Durak Online collects when you use the Service, how that information is used, who it is shared with, and how long it is kept.

## 1. Information we collect

| Category                    | Examples                                                                              | Source                                                                         |
| --------------------------- | ------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------ |
| Discord profile data        | Discord user ID, username, global name, avatar URL                                    | Discord OAuth (`identify` scope)                                               |
| Guild membership            | List of guilds you belong to (used only to determine where the activity was launched) | Discord OAuth (`guilds` scope)                                                 |
| Email + password (optional) | Email, salted+hashed password                                                         | Provided by you if you sign up with the email/password flow instead of Discord |
| Gameplay state              | Room IDs, hands, played cards, turn order, win/loss outcomes                          | Generated as you play                                                          |
| Profile + stats             | Display name, ELO, win/loss counts, achievement progress                              | Generated as you play; stored in MongoDB                                       |
| Technical diagnostics       | Error stack traces, connection events, room lifecycle events                          | Generated by the client and server                                             |

We do **not** collect: payment information, IP-based location data beyond what Fly.io and Discord receive as part of normal request routing, voice or video, contents of Discord messages outside the activity, or data from any guild you have not opened the activity in.

## 2. OAuth scopes requested

When you open Durak inside Discord, the activity requests the following Discord OAuth scopes (see `packages/client/src/discordAuth.ts`):

- `identify` — to know which Discord account is playing
- `guilds` — to know which server the activity was launched from

We never request scopes for messages, voice, friends, or DMs.

## 3. How we use information

- Authenticate you and prevent impersonation
- Match you with opponents and persist your stats across sessions
- Show your profile, leaderboard position, and achievements
- Diagnose bugs and improve game stability

We do **not** use your information for advertising, profiling, or sale to third parties.

## 4. Data retention

| Data                                                             | Retention                                                            |
| ---------------------------------------------------------------- | -------------------------------------------------------------------- |
| Active game room state                                           | In memory only; discarded when the room ends                         |
| Account profile (Discord ID, username, ELO, stats, achievements) | Kept while your account is active. Deleted on request (see §6).      |
| Email/password account credentials                               | Kept while your account is active. Deleted on request.               |
| Diagnostic logs                                                  | Retained for up to 30 days by our hosting provider, then rotated out |

## 5. Third parties we share data with

The Service relies on a small set of vendors. Each receives only what it needs to perform its function:

| Vendor                      | What it sees                                                      | Purpose                                   |
| --------------------------- | ----------------------------------------------------------------- | ----------------------------------------- |
| **Discord** (Discord, Inc.) | Your OAuth identity; any data Discord receives as a platform host | Authentication and embedding the activity |
| **MongoDB Atlas**           | Profile, stats, achievements                                      | Primary database                          |
| **Fly.io**                  | Network requests to the game server                               | Application hosting                       |
| **Sentry** (if enabled)     | Error stack traces, browser/runtime metadata                      | Crash reporting and diagnostics           |

We do not sell personal data. We do not share data with advertisers.

## 6. Deletion requests

To request deletion of your Durak profile and associated stats, email **tubulol12345@gmail.com** from the email tied to your account (or include your Discord user ID if you signed in via Discord). We will remove your profile, stats, and achievement records from MongoDB within **30 days** of a verified request, and confirm by reply.

You can also stop using the Service at any time — leaving the game and revoking the Discord application authorization (Discord → User Settings → Authorized Apps) prevents further data collection, though existing records persist until you request deletion.

## 7. Children's privacy

The Service is not directed at children under 13 (or the minimum age required by your local law, whichever is higher). We do not knowingly collect data from children below that age. If you believe a child has provided us data, email the contact above and we will remove it.

## 8. Security

- Passwords are hashed (never stored in plaintext)
- Authentication tokens are JWTs signed with a secret rotated only by the operator
- Database access is restricted by network rules to the production application
- Transport is HTTPS-only (`force_https = true` in `fly.toml`)

No system is perfectly secure; please report vulnerabilities to the contact email above.

## 9. Changes to this policy

We may update this policy. Material changes will be announced in the in-app footer link to this document and noted in the "Last updated" date at the top.

## 10. Contact

Questions, deletion requests, or vulnerability reports: **tubulol12345@gmail.com**.
